⚡ Quick Reference ▾
Asymptotic Notations

O(f(n)): upper bound (worst case)
Ω(f(n)): lower bound (best case)
Θ(f(n)): tight bound (average)
O hierarchy: 1 < logn < n < nlogn < n² < 2ⁿ

Master's Theorem

T(n) = aT(n/b) + f(n)
Case 1: f(n)=O(n^logb_a-ε) → Θ(n^logb_a)
Case 2: f(n)=Θ(n^logb_a) → Θ(n^logb_a · logn)
Case 3: f(n)=Ω(n^logb_a+ε) → Θ(f(n))

Divide & Conquer

Quick Sort: avg O(nlogn), worst O(n²)
Merge Sort: always O(nlogn), O(n) space
Divide → Conquer → Combine
Recurrence: T(n)=2T(n/2)+n → Θ(nlogn)

Complexity Classes

Linear: O(n), Log: O(logn)
Quadratic: O(n²), Cubic: O(n³)
Exponential: O(2ⁿ), Factorial: O(n!)
Polynomial: O(n^k) for constant k

Greedy Strategy

Fractional Knapsack: sort by p/w ratio
Job Sequencing: sort by profit desc, assign latest slot
Huffman: always merge 2 lowest freq nodes
Greedy choice property + Optimal substructure

0/1 Knapsack DP

V[i][w] = max profit using items 1..i, weight ≤ w
If w[i]>w: V[i][w]=V[i-1][w]
Else: max(V[i-1][w], p[i]+V[i-1][w-w[i]])
Time: O(nW), Space: O(nW)

OBST

w(i,j), e(i,j), r(i,j) tables
e(i,j) = min over r of e(i,r-1)+e(r,j)+w(i,j)
w(i,j) = w(i,j-1)+q[j]+p[j]
Base: e(i,i)=q[i], w(i,i)=q[i]

Multistage Graphs

Forward approach: cost(i,j) = min over successors
Backward: bcost(i,j) = min over predecessors
d[j] = min {c(j,l)+d[l]} for each stage
Time: O(V+E)

Backtracking

State space tree — depth-first search
Promising function checks constraints
N-Queens: check col, diag, anti-diag
Graph Coloring: no two adj nodes same color

Branch & Bound

LIFO: use stack (DFS-like)
FIFO: use queue (BFS-like)
LC: use priority queue — best first
Bound: upper bound for pruning (knapsack greedy)

TSP Branch & Bound

Lower bound: sum of row/col minimums / 2
Reduce cost matrix: subtract row min, then col min
Best solution found = upper bound
Prune if lb ≥ current best

0/1 Knapsack B&B

Upper bound = current profit + fractional fill of remaining
Sort items by p/w descending
LC search: explore node with best upper bound first
Kill node if ub ≤ best solution found

P vs NP

P: solvable in poly time (det.)
NP: verifiable in poly time (non-det.)
NP-Complete: NP + every NP reduces to it
NP-Hard: at least as hard as NP-Complete

NP-Complete Problems

3-SAT: satisfiability of 3-literal clauses
Vertex Cover: min set covering all edges
Clique: find clique of size k
Circuit SAT: first NP-Complete (Cook's theorem)

Reduction Method

To show X is NP-Complete:
1. Show X ∈ NP (poly verifier)
2. Choose known NPC problem Y
3. Show Y ≤p X (Y reduces to X in poly time)
Transitivity of reductions

Non-Deterministic Algo

Two phases: guessing + verification
ndChoice(S): guesses from S in O(1)
ndAccept/ndReject: signals result
Polynomial non-det ≡ NP class

Approximation Algorithms

Performance ratio ρ(n): cost(approx)/cost(opt)
TSP 2-approx: MST + DFS preorder walk
Works only when triangle inequality holds
No poly-time (1+ε)-approx for general TSP

Randomized Algorithms

Randomized QS: random pivot → expected O(nlogn)
Las Vegas: always correct, random time
Monte Carlo: random correct, fixed time
Expected time better than worst case

Genetic Algorithm

1. Initialize population (chromosomes)
2. Evaluate fitness
3. Selection (roulette/tournament)
4. Crossover (single/double point)
5. Mutation (flip bit)
6. Repeat until convergence

Simulated Annealing

Accept worse solution with prob e^(-Δ/T)
Temperature T decreases over time (cooling)
At T→0: acts like greedy
Avoids local minima; good for optimization

PRAM Model

EREW: exclusive read/write
CREW: concurrent read, exclusive write
CRCW: concurrent read/write
p processors, shared memory

Parallel Analysis

Speedup S(n,p) = T(n,1)/T(n,p)
Efficiency E = S/p ≤ 1
Amdahl's Law: S ≤ 1/(f+(1-f)/p)
f = serial fraction

Brent's Theorem

If algo has t steps and total ops W:
T(p) ≤ W/p + t
Key insight: work/processors + critical path
Use to bound parallel time given p processors

Parallel Patterns

Prefix sum: O(logn) steps with n/2 processors
Matrix mult: O(logn) using n³ processors
Min finding: O(logn) steps, PRAM CRCW
List ranking: pointer jumping technique
